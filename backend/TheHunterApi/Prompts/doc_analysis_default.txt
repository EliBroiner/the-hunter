# Document Analysis Prompt — Output ONLY raw JSON

Analyze the following document text and output ONLY raw JSON.

=== PRIMARY PURPOSE (CRITICAL) ===
Classify by the document's PRIMARY PURPOSE (header, main content, document type).
Do NOT classify as "Receipt" or "Invoice" just because "payment" or "total" appears in fine print or footer.

=== REQUIRED JSON STRUCTURE ===
{
  "category": "string",
  "date": "YYYY-MM-DD or null",
  "tags": ["tag1", "tag2"],
  "summary": "string",
  "metadata": {
    "names": ["string"],
    "ids": ["string"],
    "locations": ["string"]
  },
  "requires_high_res_ocr": boolean
}

=== MULTILINGUAL OUTPUT (MANDATORY) ===
If Hebrew is detected, provide "category", "tags", and "summary" in "English / Hebrew" format.
- Example category: "Invoice / חשבונית"
- Example tag: "Financial / פיננסי"
- Example summary: "Electricity bill for January / חשבון חשמל לינואר"

=== REQUIRES_HIGH_RES_OCR (mandatory boolean) ===
Set "requires_high_res_ocr" to true ONLY if the text is fragmented, garbled, or suggests a complex layout (tables/handwriting) that needs a professional scan (Google Vision). Otherwise, set to false.

=== STRICT RULE: NO PII IN TAGS ===
- The "tags" array MUST contain ONLY general categories (e.g., Receipt, Medical, Contract).
- ABSOLUTELY NO names, personal IDs, specific dates, or specific locations in tags.
- Place all detected names, IDs, and locations ONLY in the "metadata" object.

=== OUTPUT ===
Return ONLY raw JSON. No markdown, no code blocks.

=== FEW-SHOT EXAMPLES ===
Input: "Invoice #555 for John Smith, Tel Aviv"

Output: {"category":"Flight Ticket / כרטיס טיסה","date":"2024-05-20","tags":["Travel / נסיעות","Flight / טיסה"],"summary":"Flight ticket to Paris for Ariel / כרטיס טיסה לפריז עבור אריאל","metadata":{"names":["אריאל"],"ids":[],"locations":["Paris"]},"requires_high_res_ocr":false}
